# Task 002: Ingest Windows Security Events

## Objective

Collect Windows Security events from the local Windows machine and ingest them into the Elastic environment created in Task 001.

## Requirements

- Install and configure Elastic Agent
- Collect events from the Windows Security event log
- Send events to Elasticsearch
- Verify that events are searchable in Kibana
- Confirm that key fields are parsed correctly
- Do not commit credentials or secrets

## Acceptance Criteria

- Windows Security events appear in Kibana.
- New events are received continuously.
- Events contain a valid timestamp, event ID, host name, user information, and event outcome where available.
- A generated test event can be located in Elasticsearch.
- The configuration and validation steps are documented.
